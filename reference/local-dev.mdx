description: "Run a single-node Kafka 7.6.0 stack with Schema Registry, Kafka Connect, and optional Flink on your laptop using the FSI reference Docker Compose file."
---

The FSI platform ships a complete local development environment in `reference/local-dev/docker-compose.yml`. It runs Confluent Community Edition 7.6.0 containers: a single-broker Kafka cluster in KRaft mode, Schema Registry, Kafka Connect (with the JDBC connector pre-installed), and an optional Apache Flink cluster for SQL streaming development. The stack requires no Confluent Cloud account and no license. It is the recommended environment for developing against the reference producer and consumer before deploying to a cloud scenario.

<Tip>
Use this stack to validate your application, Avro schemas, and Connect configurations locally before submitting a topic onboarding request. Running `roundtrip-test.sh` against it proves your schema and produce/consume path work end-to-end.
</Tip>

## Services

The compose file defines the following services:

```yaml
services:
  broker:
    image: confluentinc/cp-kafka:7.6.0
    hostname: broker
    container_name: fsi-broker
    ports:
      - "9092:9092"
      - "9101:9101"
    environment:
      KAFKA_NODE_ID: 1
      KAFKA_PROCESS_ROLES: broker,controller
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT,HOST:PLAINTEXT
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://broker:29092,HOST://localhost:9092
      KAFKA_CONTROLLER_QUORUM_VOTERS: 1@broker:29093
      KAFKA_LISTENERS: PLAINTEXT://broker:29092,CONTROLLER://broker:29093,HOST://0.0.0.0:9092
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
      KAFKA_TRANSACTION_STATE_LOG_MIN_ISR: 1
      KAFKA_TRANSACTION_STATE_LOG_REPLICATION_FACTOR: 1
      KAFKA_JMX_PORT: 9101
      CLUSTER_ID: fsi-local-dev-cluster-001

  schema-registry:
    image: confluentinc/cp-schema-registry:7.6.0
    hostname: schema-registry
    container_name: fsi-schema-registry
    depends_on:
      broker:
        condition: service_healthy
    ports:
      - "8081:8081"
    environment:
      SCHEMA_REGISTRY_HOST_NAME: schema-registry
      SCHEMA_REGISTRY_KAFKASTORE_BOOTSTRAP_SERVERS: broker:29092
      SCHEMA_REGISTRY_LISTENERS: http://0.0.0.0:8081

  connect:
    image: confluentinc/cp-kafka-connect:7.6.0
    hostname: connect
    container_name: fsi-connect
    depends_on:
      broker:
        condition: service_healthy
      schema-registry:
        condition: service_healthy
    ports:
      - "8083:8083"
    environment:
      CONNECT_BOOTSTRAP_SERVERS: broker:29092
      CONNECT_GROUP_ID: fsi-connect-local
      CONNECT_VALUE_CONVERTER: io.confluent.connect.avro.AvroConverter
      CONNECT_VALUE_CONVERTER_SCHEMA_REGISTRY_URL: http://schema-registry:8081
    command:
      - bash
      - -c
      - |
        echo "Installing connectors..."
        confluent-hub install --no-prompt confluentinc/kafka-connect-jdbc:10.7.6
        echo "Starting Connect..."
        /etc/confluent/docker/run
```

## Service endpoints

| Service | Endpoint | Description |
|---------|----------|-------------|
| Kafka broker | `localhost:9092` | Client bootstrap server |
| Kafka JMX | `localhost:9101` | JMX metrics port |
| Schema Registry | `http://localhost:8081` | REST API for schema operations |
| Kafka Connect | `http://localhost:8083` | REST API for connector management |
| Flink UI | `http://localhost:8085` | Available with `--profile flink` |

## Starting the stack

<Steps>
  <Step title="Start core services">
    Start Kafka, Schema Registry, and Connect. Connect installs the JDBC connector on first boot (takes 30â€“60 seconds).

    ```bash
    cd reference/local-dev
    docker compose up -d
    docker compose logs -f connect
    ```

    Wait until Connect logs show `Started connector`.
  </Step>
  <Step title="Verify health">
    ```bash
    # Broker
    kafka-broker-api-versions --bootstrap-server localhost:9092

    # Schema Registry
    curl http://localhost:8081/subjects

    # Connect
    curl http://localhost:8083/connectors
    ```
  </Step>
  <Step title="(Optional) Start Flink">
    Flink is gated behind the `flink` profile. It starts a JobManager, TaskManager with 4 task slots, and an SQL client container.

    ```bash
    docker compose --profile flink up -d

    # Open Flink SQL client
    docker exec -it fsi-flink-sql-client /opt/flink/bin/sql-client.sh
    ```

    The Flink UI is available at `http://localhost:8085`.
  </Step>
  <Step title="Run the integration roundtrip test">
    ```bash
    cd reference/integration-test
    ./roundtrip-test.sh
    ```

    The test runs against the local stack by default (`localhost:9092`, `http://localhost:8081`). See the [Integration test](#integration-roundtrip-test) section for details.
  </Step>
  <Step title="Tear down">
    ```bash
    cd reference/local-dev
    docker compose --profile flink down
    # or without Flink:
    docker compose down
    ```
  </Step>
</Steps>

## Integration roundtrip test

`reference/integration-test/roundtrip-test.sh` verifies the full produce â†’ Schema Registry â†’ topic â†’ consume â†’ verify path in five steps:

```bash
#!/usr/bin/env bash
# Usage:
#   ./roundtrip-test.sh [bootstrap_server] [sr_url]
# Defaults to local-dev endpoints.

BOOTSTRAP="${1:-localhost:9092}"
SR_URL="${2:-http://localhost:8081}"
TOPIC="test.integration.v1.roundtrip"
TEST_KEY="test-$(date +%s)"
TEST_VALUE="roundtrip-$(uuidgen 2>/dev/null || echo $RANDOM)"
```

The five steps:

1. **Create topic** â€” `kafka-topics --create` with 1 partition, replication factor 1
2. **Register schema** â€” POST to `{SR_URL}/subjects/{topic}-value/versions` with the `RoundtripTest` Avro record
3. **Produce** â€” `kafka-avro-console-producer` with the test schema
4. **Consume and verify** â€” `kafka-avro-console-consumer --from-beginning --max-messages 1`, grep for the test key
5. **Verify Schema Registry** â€” confirm `{topic}-value` subject exists in the registry

The test exits 0 on success and 1 on any failure, making it safe to use as a CI step.

To run against a remote environment:

```bash
./roundtrip-test.sh pkc-xxxxx.us-east-1.aws.confluent.cloud:9092 \
    https://psrc-xxxxx.us-east-1.aws.confluent.cloud
```

## Kafka Connect JDBC configuration

The local Connect container ships with `kafka-connect-jdbc:10.7.6` pre-installed. Reference connector configs are in `reference/connect-configs/`.

<AccordionGroup>
  <Accordion title="JDBC Source connector â€” Oracle polling">

    Polls Oracle tables `TRANSACTIONS` and `MEMBER_UPDATES` every second, routing rows to topics prefixed `corebanking.core.v1.`. Credentials are injected from Vault.

    ```json
    {
      "name": "corebanking-jdbc-source",
      "config": {
        "connector.class": "io.confluent.connect.jdbc.JdbcSourceConnector",
        "connection.url": "jdbc:oracle:thin:@oracle.fsi.internal:1521/COREDB",
        "connection.user": "${vault:secret/fsi/oracle#username}",
        "connection.password": "${vault:secret/fsi/oracle#password}",
        "table.whitelist": "TRANSACTIONS,MEMBER_UPDATES",
        "mode": "timestamp+incrementing",
        "timestamp.column.name": "UPDATED_AT",
        "incrementing.column.name": "TXN_ID",
        "topic.prefix": "corebanking.core.v1.",
        "poll.interval.ms": "1000",
        "batch.max.rows": "1000",
        "value.converter": "io.confluent.connect.avro.AvroConverter",
        "value.converter.schema.registry.url": "https://schema.fsi.internal",
        "errors.tolerance": "none"
      }
    }
    ```

  </Accordion>
  <Accordion title="JDBC Sink connector â€” Oracle upsert">

    Upserts records from `corebanking.core.v1.account-balance` into `KAFKA_ACCOUNT_BALANCE` using `account_number` as the primary key. Failed records route to a DLQ topic.

    ```json
    {
      "name": "corebanking-jdbc-sink",
      "config": {
        "connector.class": "io.confluent.connect.jdbc.JdbcSinkConnector",
        "connection.url": "jdbc:oracle:thin:@oracle.fsi.internal:1521/COREDB",
        "connection.user": "${vault:secret/fsi/oracle#username}",
        "connection.password": "${vault:secret/fsi/oracle#password}",
        "topics": "corebanking.core.v1.account-balance",
        "insert.mode": "upsert",
        "pk.mode": "record_value",
        "pk.fields": "account_number",
        "auto.create": false,
        "auto.evolve": false,
        "table.name.format": "KAFKA_ACCOUNT_BALANCE",
        "value.converter": "io.confluent.connect.avro.AvroConverter",
        "errors.tolerance": "all",
        "errors.deadletterqueue.topic.name": "corebanking.core.v1.dlq-account-balance",
        "errors.deadletterqueue.context.headers.enable": true
      }
    }
    ```

  </Accordion>
</AccordionGroup>

To register a connector against the local stack:

```bash
curl -s -X POST http://localhost:8083/connectors \
  -H "Content-Type: application/json" \
  -d @reference/connect-configs/jdbc-source-example.json
```

<Warning>
The `${vault:secret/...}` credential references in the connector configs require HashiCorp Vault with the Kafka Connect Vault extension. For local development, substitute literal values. Never commit credentials to source control.
</Warning>
